The Language of Geometry

Warning

This is still unfinished. Many things that you may expect to work are not. In a random order:
  • There are no mathematical operation yet (i.e. 3*4 = ERROR, not 12)Basic operations are supported and pi=3.14159265359 (hardcoded)
  • Every float needs a dot: 4 and 4. are not equal, nor interchangeableThe syntax for numbers should be intuitive, although the backend is free to do what it wants when converting floats to int (i.e. only in the DiscreteRepeat instruction). It should do what you expect though.
  • The semantics for Undefined is not defined for the moment. At some point this will be used to describe "infinity" when drawing.
  • The naming convention will be revised. This is superficial but still.
  • This is still generally rough : the screen is limited, and so are many things.
Generally speaking, if you strictly respect the syntax, you should have no surprise (or you are welcome to report them!)

Syntax

Number ::= | 0, 1, 2, -1, 1.5, pi, ...
| Number + Number | Number - Number
| Number * Number | Number / Number
| Undefined
Program ::= | Program ; Program
| Draw(Number,Number)
| Set(Number,Number)
| Save(string)
| Load(string)
| Turn(Number)
| DiscreteRepeat(Number) { Program }
| Integrate(Number) { Program }
| {}
